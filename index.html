About

After a long time I decided to make this project public.

The first version was only for administration tables, not enough to be a good admin tool.

But the two tools have now reached maturity and can manage very well databases.


The purpose of this program is to ease the programmer's work.

Both tools are very fast and small, and easy to work with them.

Next releases

EdLiteAdmin version 3.0 will be with sqlite3 and pdo_sqlite.

EdMyAdmin


EdMyAdmin is a faster full alternative for phpmyadmin.

Web based mysql admin tool.
Only one file ~80KB.
Small, fast, and contain all necessary operation for manage mysql.

Download: ZIP TAR

Requirements:

  • php 4.3.0 to 5.6.x
  • php mysql extension
  • mysql 4.1.x or greater
  • jQuery 1.3 or greater

Settings:

Settings are optional! (search variable at the top of the file)

  • $step => rows on page (default: 15)
  • jQuery => default: from official site

Features:

  • CRUD for database, table, view, trigger, routine, event
  • Copy table in another database
  • Create, edit, update, delete view and trigger
  • Change table name (changing table name in view and in trigger)
  • Change a field from table (changing field name in view and in trigger if exist)
  • Drop the field from table (droping view if contain that field)
  • Delete the table (delete associated views and triggers)
  • Import SQL, CSV
  • Export SQL, CSV with optional pack GZ or ZIP
  • User management
  • Flash messages in the right corner
  • Running with mod rewrite enabled
  • And much more...

Rules:

CSV Import

  • CSV file name must be the same as the table name
  • In first row must be defined the table fields name

EdLiteAdmin


Web based sqlite3 admin tool.
Only one file ~45KB.
Small, fast, and contain all necessary operation for manage sqlite.

Download: .ZIP .TAR

Requirements:

  • php 5.3.0 or greater
  • php sqlite3 extension
  • jQuery 1.3 or greater

Settings:

Settings are optional! (search variable at the top of the file)

  • $ext => sqlite database extension (default: .db3)
  • $dir => sqlite database folder (default: current folder)
  • $step => rows on page (default: 15)
  • jQuery => default: from official site

Features:

  • When creating the database is not required extension
  • CRUD for database, table, view, trigger
  • Copy table in another database
  • Create, edit, update, delete view and trigger
  • Change table name (changing table name in view and in trigger)
  • Change a field from table (changing field name in view and in trigger if exist)
  • Drop the field from table (droping view if contain that field)
  • Delete the table (delete associated views and triggers)
  • Import SQL, CSV
  • Export SQL, CSV with optional pack GZ or ZIP
  • Flash messages in the right corner
  • Running with mod rewrite enabled
  • And much more...

Rules:

CSV Import

  • CSV file name must be the same as the table name
  • In first row must be defined the table fields name
