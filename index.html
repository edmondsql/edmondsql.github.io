About

After a long time I decided to make this project public.

The first version was only for tables administration, not enough to be a good admin tool.

But the two tools have now reached maturity and can manage very well databases.


The purpose of this program is to ease the programmer's work.

Both tools are very fast, small, and easy to use.

EdMyAdmin


EdMyAdmin is a faster full alternative for phpmyadmin.

Web based mysql admin tool.
Only one file ~100KB.
Small, fast, and contain all necessary operation for manage mysql.

Download: ZIP TAR

Requirements:

  • edmyadmin version 3.x work with mysqli and pdo_mysql
  • php 5.3.0 or greater
  • php mysqli and/or pdo_mysql extension
  • mysql 5.1.30 or greater
  • jQuery 1.6 or greater

edmyadmin version 2.x work only with mysql extension; php 4.3.0 to 5.6.x; mysql 4.1.x or greater

Settings:

Settings are optional!

  • jQuery => default: from googleapis (download jquery for offline use) [in $head variable]

Features:

  • CRUD for database, table, view, trigger, routine, event
  • Copy table in another database
  • Create, edit, update, delete view and trigger
  • Change table name (changing table name in view and in trigger)
  • Change a field from table (changing field name in view and in trigger if exist)
  • Drop the field from table (droping view if contain that field)
  • Delete the table (delete associated views and triggers)
  • Import sql, csv, gz, zip
  • Export sql, csv with optional pack gz or zip
  • User management
  • Flash messages in the right corner (It appears for 7 seconds; with double click on message disappear)
  • Delete with confirm message
  • Check version in info page
  • Running with mod rewrite
  • And much more...

Rules:

CSV Import

  • CSV file name must be the same as the table name
  • In first row must be defined the table fields name

GZ and Zip Import

  • File name format: [filename].[format].[extension]

filename: only for csv need to be same with the table name
format: sql or csv
extension: gz or zip

EdLiteAdmin


Web based sqlite3 admin tool.
Only one file ~60KB.
Small, fast, and contain all necessary operation for manage sqlite.

Download: .ZIP .TAR

Requirements:

  • php 5.3.0 or greater
  • php sqlite3 and/or pdo_sqlite extension
  • jQuery 1.3 or greater

Settings:

Settings are optional!

  • $dir => sqlite database folder (default: current folder) [ED class variable]
  • $ext => sqlite database extension (default: .db3) [ED class variable]
  • $passwd => login password (default: empty) [ED class variable]
  • jQuery => default: from googleapis (download jquery for offline use) [in $head variable]

Features:

  • When creating the database is not required extension
  • CRUD for database, table, view, trigger
  • Copy table in another database
  • Create, edit, update, delete view and trigger
  • Change table name (changing table name in view and in trigger)
  • Change a field from table (changing field name in view and in trigger if exist)
  • Drop the field from table (droping view if contain that field)
  • Delete the table (delete associated views and triggers)
  • Import sql, csv, gz, zip, sqlite/db3
  • Export sql, csv with optional pack gzor zip
  • Flash messages in the right corner (It appears for 7 seconds; with double click on message disappear)
  • Delete with confirm message
  • Check version in info page
  • Running with mod rewrite
  • And much more...

Rules:

CSV Import

  • CSV file name must be the same as the table name
  • In first row must be defined the table fields name

GZ and Zip Import

  • File name format: [filename].[format].[extension]

filename: only for csv need to be same with the table name
format: sql, csv, sqlite or default database extension
extension: gz or zip
